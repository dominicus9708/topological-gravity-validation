Eta Carinae topological pipeline proxy rationale\n================================================\n\nPurpose\n-------\nThis file records why component-row restructuring was introduced.\nThe goal is not to fit a preferred answer, but to make the same\ncomponent row carry mass + volume + velocity so that a structural\ntopological quantity can be computed at all.\n\nRules applied\n-------------\n1. homunculus_main_lobes\n   - if direct mass is missing, link to Great Eruption lower-bound mass proxy.\n2. dust_torus\n   - if geometry/depth/velocity are missing, inject conservative torus proxies.\n3. outer_ejecta_context -> outer_ejecta_component\n   - promote context-only row into a calculable outer-ejecta component row.\n\nApplied per-row flags\n---------------------\n- homunculus_main_lobes: homunculus_mass_linked_to_great_eruption_lower_bound\n- dust_torus: dust_torus_major_axis_proxy|minor_axis_proxy|depth_proxy|bulk_velocity_proxy\n- outer_ejecta_component: outer_ejecta_context_promoted_to_calculable_component
